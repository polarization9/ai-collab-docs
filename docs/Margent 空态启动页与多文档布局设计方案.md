# Margent 空态启动页与多文档布局设计方案

## 1. 设计目标

这次重设计解决两个核心问题：

- 空态启动页需要像真实桌面 App 的起点，而不是一个居中的介绍页。
- 多文件 tab 需要成为全局 App 承载层，而不是混在当前文档内容里的一个组件。

设计基准来自根目录 `PRODUCT.md` 和 `DESIGN.md`：Margent 是本地 Markdown 审阅与 Codex 协作工作台，视觉重点是文档优先、工具克制、状态清楚。

## 2. 空态启动页

### 2.1 页面结构

空态使用纵向单栏布局：

```text
Margent app canvas
├── 打开文档 command
└── 最近文档 list
```

- 打开文档 command 位于页面中部偏上，是唯一主操作。
- 最近文档 list 位于主操作下方，作为辅助入口，不使用额外容器包裹。
- 打开区域和最近文档列表保持同一视觉中线。
- 不再出现内嵌 macOS 窗口边框、三色圆点或大面积装饰。
- 设置入口保持全局属性，不和打开文件主任务竞争。

### 2.2 默认空态：无最近文档

视觉：

- 页面中部放置一行打开命令，包含文件图标、标题和主按钮。
- 最近打开标题固定显示在主操作下方。
- 无最近文档时，只保留轻量空状态，不额外占用大块容器。
- 主标题控制在产品界面字号，不使用 hero 级大字。

文案建议：

- 标题：打开一份 Markdown 文档
- 说明：支持 `.md` 和 `.markdown`，批注会保存在文档同目录。
- 主按钮：打开 Markdown 文件

### 2.3 有最近文档

最近文档行展示：

- 文件名。
- 最近打开时间。
- 移除按钮，hover 后显示或增强。

交互：

- 整行都是点击热区，点击行打开文件。
- 文件名溢出使用单行省略。
- path 作为 tooltip，不在默认列表中显示，避免把行撑高。

### 2.4 最近文档缺失

当文件不存在：

- 文件行保留，但主点击禁用。
- 行内展示“文件不存在”或“Missing file”。
- 移除按钮仍可用。
- 不弹窗，不把整个空态切到错误页。

### 2.5 启动恢复 / 加载中

用于 App 刚启动、sidecar 正在准备、恢复上次文档时：

- 主打开命令区域展示轻量 skeleton 或“正在恢复文档”状态。
- 最近文档列表可以先展示 skeleton 行。
- 不应先闪出 “Unable to load document”。
- 只有明确失败后才进入打开失败状态。

### 2.6 打开失败

打开文件失败时：

- 用户仍留在空态页。
- 主按钮下方展示短错误文案。
- 错误文案不超过两行。
- 提供重新打开文件的同一个主按钮，不额外堆一组操作。

### 2.7 窄屏

小于约 860px：

- 主打开命令在上，最近文档在下。
- 最近文档行保持紧凑，避免横向滚动。

## 3. 多文件布局

### 3.1 总体结构

打开文档后，布局层级应为：

```text
App
├── 全局文档 tab rail
└── 当前文档 workspace
    ├── TOC
    └── Document pane
        ├── 阅读 / 编辑内容
        └── 批注面板
```

关键原则：

- tab rail 是全局层，位于 TOC 和正文上方。
- TOC、正文、批注面板都属于当前 active document。
- 切换 tab 时，TOC、正文滚动状态、批注列表和 Codex 连接状态都切换到对应文档。

### 3.2 全局 tab rail

视觉规格：

- 高度约 52-56px。
- 顶部 sticky。
- 背景使用 app canvas 的轻透明版本，底部一条细分割线。
- 左侧为横向滚动的 tab 列表。
- 右侧为固定的 icon-only 打开文件按钮。

不要：

- 不要在 tab rail 上放编辑 / 批注按钮。
- 不要把 tab rail 放进 document pane 的宽度约束里。
- 不要使用 faux macOS 顶栏或窗口边框。

### 3.3 Tab 样式

每个 tab 包含：

- 状态点或文件图标。
- 文件名。
- 关闭按钮。

Active tab：

- 白色或强纸面背景。
- 文字使用 strong ink。
- 1px strong border。
- 无大阴影。

Inactive tab：

- 透明或 soft canvas 背景。
- muted 文本。
- hover 时背景略亮、边框出现。

Close 按钮：

- 默认弱显示。
- hover / focus-within 时增强。
- 点击范围稳定，不让文件名突然跳动。

### 3.4 打开文件按钮

右上角打开文件按钮：

- 只保留轻 `+` 或 file-plus icon。
- 36px 方形。
- 固定在 tab rail 右侧，不随 tab 横向滚动。
- tooltip 展示“打开 Markdown 文件”。

### 3.5 Tab 状态预留

后续可支持这些状态，但不在无数据时伪造展示：

- 未保存：文件名旁小点或 tab 状态点变化。
- 外部更新：tab 上轻量提示点。
- 文件缺失：tab 状态点转 warning，并禁用继续编辑。
- 批注未处理数量：小型 badge，靠近文件名右侧。

### 3.6 多文件与当前文档操作区

编辑、保存、退出编辑、批注入口、当前文档 Codex 连接状态，仍属于当前文档：

- 放在 document pane 内右上角的浮动操作区。
- 不进入全局 tab rail。
- 切换文档时跟随 active document 切换。

## 4. 实现建议

建议按以下顺序实现：

1. 先调整 DOM 层级：确保 `DocumentTabs` 是 `app-root` 下的全局兄弟节点，而不是 document pane 的子节点。
2. 重写空态 DOM：去掉 faux window 结构，保留打开命令和最近打开列表。
3. 重写 tab CSS：active / inactive / close / pinned open button / overflow。
4. 增加空态子状态样式：no recent、missing file、loading/restoring、open failed。
5. 做浏览器截图检查：空态无 recent、有 recent、多 tab、窄屏、多 tab 横向溢出。

## 5. 验收标准

- 空态页面没有内嵌 macOS 窗口边框或三色圆点。
- 空态主操作清楚，最近文档不抢主操作。
- 打开失败不会整页跳成错误中心态。
- 多文件 tab 横跨全局宽度，不受正文最大宽度影响。
- TOC 与正文都位于 active document workspace 内。
- 右上角打开文件是 icon-only，且固定在 tab rail 右侧。
- 编辑和批注按钮仍在文档内部右上角，不进入 tab rail。
